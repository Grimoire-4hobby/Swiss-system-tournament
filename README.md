# Swiss-system-tournament
HTML script for pairing in Swiss-system tournaments (can also be used for tournaments other than HPMA)  
https://note.com/ictlink/n/nd24f2b3d0492 様のスイスドロースクリプトをプロトタイプとして使用させていただきました．

## 使用方法

### PC  
index.htmlをダウンロードし，実行するかChromeなどのブラウザにドラッグすることで起動します．
### スマホ  
HTMLファイルを直接実行することはできないので，https://grimoire-4hobby.github.io/Swiss-system-tournament/ をブラウザで開き，適宜ホーム画面に追加してください．
<br>
<br>
ブラウザのlocalStorageで終始するためオフラインでも実行できます．  
原則ブラウザを閉じても削除されませんが，キャッシュ削除やストレージ限界，プライベートモードなどでは残存しない場合もあるため，重要な大会データは適宜画面下部の「JSONインポート/エクスポート」を利用してください．

## 使い方

### 1. 準備
1. 大会名を設定します．
2. 勝ち点などのオプションを設定します．デフォルトでは勝ち点３，引分（があれば）１に設定されています．また，BonusとReliefの適用にチェックを入れると勝敗の入力でBonusやReliefを選ぶことができます．  
デフォルト設定だとBonusは勝者の勝ち点３に加え１の追加点を，Reliefは敗者に１の救済点を加算します．例えばHPMAの試合が1:30以内で決した場合Bonusを，3:30以上長引いた場合Reliefを追加するといった運営を想定しています．
3. 任意で大会ロゴ画像を入れることで，各ラウンドのスコアボード出力で透けた選択画像が背景となります．
4. プレイヤー名を1行ずつ入力して「大会開始」を押します．（リセットや新規大会を同じブラウザで行う場合，再開時に以前のプレイヤー名が引き継がれ再入力する必要はありません．）

### 2. ラウンド進行
1. 生成されたペアリングに従って対戦を行います．下部の「順位表を画像で保存」ボタンを押すと，前のラウンドまでのスコアボードがpng画像でダウンロードできます．  
ラウンド時に棄権者が出た場合，下部のプレイヤー順位で該当プレイヤーの危険ボタンを押してから，再度「ペアリング生成」ボタンを押すことで棄権者を除いて再度マッチングします．また棄権者は後のラウンドで同様に復帰させることもできます．
2. 勝者の名前をタップして結果を入力します（引き分け・Bonus/Reliefも同様）．
3. 全結果を入力後「次ラウンドへ進む」を押します．

### 3. 終了と発表
1. 数学的な必要ラウンド（$\lceil \log_2 n \rceil$）が終了すると、警告と「最終結果を発表」ボタンが表示されます．  
このラウンド以降も続行することはできますが，以前のラウンドですでに対戦したペアが再戦する確率が上がります．
2. 大会を終了すると、下位から順にランクが発表される演出が始まります．  
スコア的に同率の場合OMWP（Opponent Match Win Percentage）やOOMWP（Opponent OMWP）で内部的にタイブレーカー処理が行われます．  
スコアボードpng画像では同率表記になっていますが，内部順位順に上から表示されています．
3. 最後に全順位表が表示されます．

## 仕様  

### マッチング  
標準的なスイスドローマッチングに従い，勝ち点が近いプレイヤー同士で対戦し，すでに対戦したプレイヤーとはなるべく再戦しないようになっています．  
同率順位をランダムソートして20探索のマッチングを行い，対戦ペアの点差合計と再戦が発生した場合１試合につき３点差相当のペナルティを付与し，これらの総計が最小となるマッチングを採用します．  
再戦をより排したい場合，handleGeneratePairings()内の再戦ペナルティの重みであるRE_MATCH_PENALTY = 3; を変更することで再戦発生の可能性が下がりますが，数学的な必要ラウンド数（$\lceil \log_2 n \rceil$）を超えた場合，デッドロック回避のためやむを得ず再戦が避けられない場合があります．  

また，総プレイヤー数が奇数の場合不戦勝のプレイヤーは「これまで不戦勝になっていないプレイヤーのうち最もスコアの低いプレイヤー」が優先して不戦勝シードになります．  

### Opponent Match Win Percentage（OMWP）  
OMWPはこれまでの対戦相手の勝率平均を表し，「自分がどの程度強い相手と対戦してきたか」の指標として，スコアが同点だった場合に内部的なタイブレーカー処理として高い方のプレイヤーを上位とします．nラウンド目にてこれまでの対戦相手のnラウンド時の勝率（k勝n-k敗の場合k/n）の平均をOMWPとして算出しますが，極端な勝率の影響を排するため勝率の下限を25％，上限を75％として算出します．  
さらに対戦相手全員のOMWP平均をOOMWP（Opponent Opponent Match Win Percentage）とし，OMWPまで同率の場合OOMWPの高い方を上位とします．  

### スコアボード  
各ラウンド時に「順位表を画像で保存」を押すことで，前のラウンドまでの結果スコアボードをpngで出力できます．  
その際下位n人を匿名化するかを設定できますが同率順位のプレイヤーは全員匿名化する都合上，この人数はnを超える場合があります．  
スコアボードにはRank，Player名，Score，Promotion（前ラウンドの加算点），OMWP，Status（参加or棄権）が表記され，棄権者は匿名化されません．  
また，最初のオプション選択時に任意で大会ロゴ画像を入れることで，透けた選択画像が背景となります．  
この画像はスコアボードの縦横比に応じてトリミング，リサイズされるため，参加人数によっては背景画像が見切れる場合があります．また背景の濃さはexportStandingsImage()内のctx.globalAlpha = 0.07; から変更できます．
